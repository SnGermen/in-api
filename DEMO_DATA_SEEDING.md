# Демо-данные для Instagram-клона

## Цели сидинга
- Получить фейковых пользователей (публичные/приватные, аватар опционально)
- Фейковые посты: изображения и подписи
- Лайки к постам
- Комментарии к постам
- Подписки (follows)
- Stories (с истечением 24 часа)
- Админ-пользователь для админ-панели

## Объемы данных
- Пользователи: 20 обычных + 1 админ
- Посты: 5–12 на каждого пользователя
- Комментарии: 0–20 на каждый пост
- Лайки: 0–100 на каждый пост
- Подписки: 3–10 у каждого пользователя
- Stories: 0–5 на пользователя

## Ассеты изображений
- Папка: `seed_assets/`
- Базовые файлы: `demo1.jpg`, `demo2.jpg`, `demo3.jpg`
- Дополнительно — 5–10 простых заглушек (пустые `.jpg` допустимы)
- Во время сидинга изображения копируются в `uploads/images/demo/`

## Настройка Prisma Seed Script
- Файл: `prisma/seed.ts`
- Использует `@prisma/client`, `@faker-js/faker`, `bcrypt`
- Локальная загрузка изображений: копирование из `seed_assets/` в `uploads/images/demo/`
- Создание сущностей:
  - Пользователи (включая админа)
  - Посты и связанные медиа (`Media`, `PostMedia`)
  - Лайки (`PostLike`) и комментарии (`Comment`)
  - Подписки (`Follow`)
  - Stories (`Story`, `StoryMedia`)

## Запуск
- Генерация клиента: `yarn prisma:generate`
- Миграции: `yarn prisma:migrate`
- Сидинг: `yarn prisma:seed`
- Чистый сидинг: `yarn db:reset` (reset → migrate → seed)

## Примечания
- Сидинг строго типизирован на TypeScript
- Бизнес-логика не изменяется, данные пишутся напрямую через Prisma
- Пути медиа хранятся относительно каталога `uploads/` (статически доступны по `GET /uploads/...`)
